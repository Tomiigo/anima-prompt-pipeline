# プロンプト(ルール)を編集・共有する

このフォルダには、**プログラムを触らずに**ふつうのテキストとして編集できるファイルが入って
います。ここを書き換えると、Gemma への指示が変わり、出てくる Anima プロンプトの傾向が変わります。
保存して次に `run.py` を実行すれば反映されます(作り直しや再起動は不要)。

- **`anima_rules.txt`** … Anima 用のルールを**言葉で説明する**ファイル。「どんなタグを選び、
  どう英文にするか」を文章で書きます。**いまツールが読み込んで使うのは、このファイルです。**
- **`fewshot.txt`** … ルールの結果を**具体例で見せる**ファイル。「こういう依頼には、こう答えて
  ほしい」という入力と理想の答えの組を並べます。言葉で説明しづらいこと(複数キャラの色うつりなど)は、
  良い例を 1 つ見せるほうがよく効きます。書き方は下記「`fewshot.txt` の書き方」を参照。

> 「Anima プロンプト」とは何か、ツール全体がどう動くかは、リポジトリ直下の `README.md` を
> 先に読むと分かりやすいです。

---

## ルールが、どう使われるか

`run.py` に日本語のプロンプトを渡すと、ツールは次の順で材料を組み立てて Gemma に渡します。

1. **ルール** … `anima_rules.txt` の中身を「守ってほしい指示」として渡します
   (タグをどんな形で出力するか、という最終フォーマットはプログラムが自動で付け足すので、
   ここには書きません)。
2. **お手本** … `fewshot.txt` の各例を「過去の良いやり取り」として見せます(`USE_FEWSHOT` が有効なとき)。
3. **あなたのプロンプト** … 日本語を英語に訳した依頼文を渡します。

Gemma が書いたタグは、このあとツールが辞書と照らし合わせて、実在する正規のタグへ自動で
直します(綴り違いや言い回しのズレを補正します)。つまり**ルールで方針を伝え、細かな表記は
あとで機械的に整える**、という分担です。

---

## `anima_rules.txt` の書き方

ここには「どんなタグを選ぶか」「どう英文にするか」**だけ**を書きます。

### 通し番号は付けません(共同編集のため)

ルールは**セクション(見出し)ごと**に分かれ、各行は先頭に `-` を付けた箇条書きです。
**通し番号は付けません。** 大勢で改良するときに「番号の振り直し」で衝突しないためです。

- 箇条書きは**自由に追加・削除・書き換え**してよい(セクション内の順番は問いません)。
- ルール同士で参照するときは、番号ではなく**セクション名**で指します
  (例:「TAG ORDER のとおりに並べる」)。
- Gemma は番号がなくても内容を理解します。

### 言語:説明は日本語で書けます(一部は英語のまま)

Gemma は多言語を理解するので、**ルールの説明文は日本語で書けます**。ただし次のものは、
言語に関係なく**英語のまま**にします(これらは Gemma がそのまま出力に使う「具体的な文字列」だからです)。

- タグ名・タグの例(`masterpiece`, `1girl`, `serafuku`, `@kantoku`, `score_7` など)
- 記号の決まり(アーティスト名に `@` を付ける、`(tag:1.3)` で強調する、など)
- 良い例・悪い例として見せる**英文**(出力は英語のタグ+英文なので、手本も英語で示します)

### 守ること(壊さないために)

- **出力の最終フォーマットは書かない。** タグをどんな形で出すか(プログラムが内部で決める形式)は
  **自動で付け足されます**。ここに書くと二重指定になって壊れます。このファイルには方針(ルール)だけを書きます。
- **変更履歴をファイルに書かない。** この中身はそのまま Gemma に渡るためです。「いつ何を変えたか」は、
  (GitHub を使うなら)コミットメッセージや別のメモに残してください。

## `fewshot.txt` の書き方

Gemma に見せる**お手本**です。「この説明文には、こういうタグの行(と、必要なら英文)を返す」という
入力と理想の答えの組を並べます。**例と例は `---`(ダッシュ 3 つだけの行)で区切ります**。
`#` で始まる行はメモで、無視されます。

### 1 つの例は、3 つのラベルでできています

- **`DESCRIPTION`** … Gemma に渡す**英語の説明文**(あなたの日本語プロンプトを訳したものがここに入ります)。
- **`TAGS`** … Gemma に**出してほしいタグの行**(カンマ区切り)。並び順は `anima_rules.txt` の
  **TAG ORDER** のとおり:品質/メタ/レーティング → 人数(+相互作用) →
  キャラ名・作品名・`@`アーティスト → 容姿(髪→目→表情→服→小物) → ポーズ →
  カメラ → 背景/時間/天気 → ライティング/効果。
- **`NATURAL`** … **任意**。場面が複雑なときだけ、英文を**1 文**書きます。空でよい。
  代名詞(she / he / they / it)は自然文では**使ってよい**(タグ側には書きません)。

`DESCRIPTION` と `TAGS` だけは**必須**です(この 2 つが無いかたまりは読み飛ばされます)。

### 新しいお手本の足し方

1. 既存の例を 1 つ選び、`---` から `---` までの**かたまりをまるごとコピー**します。
2. コピーした中身を、自分の例に書き換えます。
3. 例と例の間に `---` の行が残っていることを確認します。

### 書くときのコツ

- **英語で書く**(出力が英語のため)。説明文(`anima_rules.txt`)は日本語にできますが、お手本は英語で示します。
- `TAGS` は **TAG ORDER の並び**で。アーティストは `@` 付き、キャラ名のあとに作品名を置きます。
- 各キャラの容姿は、**そのキャラを言ったすぐ後**に、髪→目→表情→服→小物の順で書きます。
- **複数キャラの色うつり対策**:`NATURAL` の 1 文で「左の子は…、右の子は…」と割り当てると安定します
  (同梱の 2 例目がその実演です)。
- **無い名前をでっち上げない**:キャラ名・アーティスト名は、説明文に出てきたときだけ書きます。

### おすすめの構成

単一キャラ / 色うつりのある複数キャラ / アーティスト指定 / キャラ名つき / 「悪い例→直した例」など、
**4〜6 例**。数より**パターンの一貫性**が大事です。同梱の `fewshot.txt` に 5 例入っているので、
まずはそれを土台に増減してください。

---

## 改良の進め方(人間が教える役)

1. 生成して、画像を見る。
2. イメージとズレた点を**1 つだけ**選ぶ(例:「複数人のとき目の色がうつった」)。
3. それに効きそうなルールを**1 つだけ**直す。
4. **同じ入力**で生成し直して見比べる。良くなったら採用、ダメなら戻す。
5. 安定して効いた変更だけを、共有版に反映する。

一度に複数を変えると、どれが効いたか分からなくなります。**1 回 1 箇所**が鉄則です。

---

## 編集したあと

- **特別な構文チェックは要りません**(ふつうのテキストです)。`fewshot.txt` は、各例に
  `DESCRIPTION` と `TAGS` があり、例の間に `---` が残っていれば大丈夫です。
- **反映**:保存して、次に `run.py` を実行するだけ。作り直しも再起動も要りません。
- **共有**:`anima_rules.txt` と `fewshot.txt` を渡せば完結します。変更履歴は、ファイル本文ではなく、
  コミットメッセージやメモに残してください。
