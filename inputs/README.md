# inputs/ — 変換したい文章を置く場所

ここに、**変換したい日本語の文章を書いたテキストファイルを置いてください。**

ルールは2つだけです。

1. **ファイル名は何でも構いません。** `scene.txt` でも `お姫様だっこ.txt` でも、好きな名前で大丈夫です。
2. **`.txt` ファイルは、ここに1つだけ置いてください。** パイプラインは、この `inputs/` の中にある `.txt` を自動で1つ拾います。複数あるとどれを使うか決められないため、エラーで止まります（この `README.md` は対象外なので置いたままで大丈夫です）。

> 「1つだけ」と言っても、**毎回おなじファイルを使い回す必要はありません。** シーンごとに新しいファイルを作っても構いません。要は「実行する瞬間に、ここに `.txt` が1つだけある」状態にしてください。前のファイルは消すか、`inputs/` の外に移動しておけば OK です。

置いたら、リポジトリのルートで:

```bash
./anima_pipeline_oneshot.sh
```

を実行すると、ここにある文章が変換され、ルートに `output.txt`（最終プロンプト）が出力されます。

なお、ここに置いた入力ファイルと出力（`output.txt`）は、利用者ごとに中身が異なるため **Git の追跡対象外**（`.gitignore` 済み）です。この `README.md` だけが追跡されます。
